### 1. 为什么“边学边做”没问题？

C++ 的核心语法非常庞大（C++11/14/17/20/23），有些特性（如模板元编程）甚至连写了十年代码的人也未必精通。

*   **语法的本质**：异常处理的语法其实非常直观。你只需要理解它的**控制流**：
    *   `try`: “我试着跑这段代码。”
    *   `throw`: “坏了，出大事了，我不干了，谁能处理谁接手。”
    *   `catch`: “别慌，我来处理这个烂摊子。”

### 2. 快速补课：三分钟掌握 C++ 异常核心

为了让你重构 `createUser` 时不心虚，你只需要记住这几个关键点：

1.  **抛出 (Throw)**：
    ```cpp
    if (ptr == nullptr) throw std::runtime_error("空指针异常");
    ```
    一旦 `throw` 执行，函数会**立即停止**，像坐电梯一样直接跳到最近的 `catch` 块。

2.  **捕获 (Catch)**：
    *   `catch (const std::exception& e)`：捕获标准错误。通过 `e.what()` 可以拿到错误描述字符串。
    *   `catch (...)`：这是“大网”，捕获一切。在银行系统里，它用来做最后的清理（如 `db.rollback()`）。

3.  **RAII（资源获取即初始化）—— 架构师的秘密武器**：
    在 C++ 中，如果函数因为异常退出了，函数内部的局部对象（如 `QString`）会自动调用析构函数释放内存。这就是为什么我们不需要像 Java 或 C# 那样频繁使用 `finally`。
