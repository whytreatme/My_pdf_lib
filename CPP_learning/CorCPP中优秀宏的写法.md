## 问  
我不明白为什么日志的那个宏要放在只执行一次的do while循环里
```c
#define LOG(...) do { \
        char time_buf[64]; \
        time_t now = time(0); \
        strftime(time_buf, sizeof(time_buf), "%Y-%m-%d %H:%M:%S", localtime(&now)); \
        printf("[%s] [tid:%ld] ", time_buf, syscall(SYS_gettid)); \
        printf(__VA_ARGS__); \
        printf("\n"); \
        fflush(stdout); \
    } while (0)
```

## 答



这是一个非常经典的问题！你能注意到这个细节，说明你看代码非常仔细。

在 C/C++ 中，把宏定义包在 `do { ... } while (0)` 里，被称为 **“宏定义的最佳实践”** 或 **“工业级宏写法的标配”**。几乎所有优秀的开源 C/C++ 项目（包括 Linux 内核源码）在写多行宏时，都会强制使用这种格式。

这也是 C/C++ 面试中极高频的一个考点。

我们不用干瘪的理论，直接用代码来“破案”，看看如果不这么写，会发生什么可怕的灾难。

---

### 灾难现场一：不用任何大括号

假设我们写了一个最简单的、不带任何括号的宏：

```cpp
#define LOG()  printf("第一行\n"); printf("第二行\n")
```

看起来没问题对吧？现在我们在一个单行的 `if` 语句中使用它：

```cpp
if (condition)
    LOG();
```

**宏展开后，变成了这样：**

```cpp
if (condition)
    printf("第一行\n"); 
printf("第二行\n");    // ！！！灾难发生！！！
```

由于 `if` 后面没有大括号，它只管得住第一句话。结果就是：**无论条件是否成立，“第二行”永远会被打印！** 这绝对不是我们想要的结果。

---

### 灾难现场二：只用普通的 `{}` 括起来

既然不用括号不行，那我们像写普通作用域一样，只用花括号 `{}` 把它包起来行不行？

```cpp
#define LOG() { printf("第一行\n"); printf("第二行\n"); }
```

普通的 `if` 语句确实没问题了，但是当我们遇到 **`if-else` 结构** 时，致命语法错误就出现了：

```cpp
if (condition)
    LOG();
else
    do_something();
```

由于程序员在使用函数或宏时，**习惯性地会在末尾加一个分号（`;`）**，宏展开后变成了这样：

```cpp
if (condition)
    { printf("第一行\n"); printf("第二行\n"); };   // <--- 注意这个分号！
else                                             // <--- 编译报错：'else' without a previous 'if'
    do_something();
```

**编译直接报错！** 
因为那个多出来的分号 `;` 被编译器当成了一个空语句。编译器看到的情况是：
1. 执行 `if` 块里面的代码。
2. 执行一条空语句（`;`）。`if` 语句到这里彻底结束了。
3. 突然冒出来一个 `else`，编译器懵了：“你的 `if` 呢？”

---

### 终极救星：`do { ... } while (0)`

为了让宏在任何情况下表现得**完完全全像一个普通的函数调用语句**，前辈大牛们发明了 `do { ... } while (0)`。

```cpp
#define LOG() do { printf("第一行\n"); printf("第二行\n"); } while(0)
```

我们再把刚才导致编译报错的 `if-else` 代码拿来展开：

```cpp
if (condition)
    LOG();
else
    do_something();
```

**完美展开后：**

```cpp
if (condition)
    do { printf("第一行\n"); printf("第二行\n"); } while(0);  // <--- 完美吸收了末尾的分号！
else
    do_something();
```

**奇迹发生了：**
1. `do-while` 循环本身就是一个单一的 C 语言语句，它完美地把多行代码打包在了一起。
2. `while(0)` 保证了这段代码**绝对只会执行一次**，没有任何性能开销。
3. 最精妙的是：`do-while` 语法**强制要求在 `while()` 后面必须跟一个分号！** 所以程序员在调用 `LOG();` 时习惯性加的那个分号，恰好成为了 `do-while` 语句合法结束的标志。它完美地“吞噬”了那个分号，不会导致 `else` 孤立！

### 附加好处：干净的局部作用域

除了解决分号问题，`do { ... } while (0)` 还自带一对大括号 `{}`，这意味着它创建了一个**独立的局部作用域**。

看你代码里的这一段：
```cpp
#define LOG(...) do { \
    char time_buf[64]; \
    time_t now = time(0); \
    // ...
} while(0)
```
因为在这个独立的局部作用域里，你定义了 `time_buf` 和 `now` 这两个局部变量。当宏执行完毕（跳出 `while(0)`），这两个变量就立即销毁了，**绝对不会和调用宏的外部函数里可能存在的同名变量发生冲突**。

这就是为什么底层 C/C++ 程序员对 `do { ... } while (0)` 爱不释手的原因，它是 C 语言宏编程中最优雅、最严谨的一块补丁。
