### 你的 Qt Widgets 工业级实践复盘清单

#### **第一部分：Qt 核心对象模型 (The Soul of Qt)**

**第1-3课: QWidget, QDialog, QMainWindow**
*   **核心知识点**:
    1.  `QWidget` 是所有UI控件的“祖宗”。
    2.  `QDialog` 专用于做弹出对话框，有“模态”(`exec`)和“非模态”(`show`)两种模式。
    3.  `QMainWindow` 是带有菜单栏、工具栏、状态栏的复杂“精装房”主窗口。
*   **工业级认知**:
    1.  **内存管理**: `new QWidget(this)` 把 `this` 当作父对象，Qt 对象树会自动回收内存（“拔萝卜”机制），让你告别手动 `delete` 的噩梦。
    2.  **模态陷阱**: `exec()` 会阻塞代码，如果在主窗口的**构造函数**里调用，主窗口会“难产”而无法显示，直到弹窗关闭。这是新手必踩的坑。
    3.  **布局规则**: `QMainWindow` 有自己的固定布局，不能直接 `setLayout()`，必须用 `setCentralWidget()` 设置核心工作区。

**第4-5课: QObject, 坐标系, QDebug**
*   **核心知识点**:
    1.  `QObject` 是对象树和信号槽的基石。
    2.  Qt 坐标系原点在左上角，Y 轴向下。
    3.  `qDebug()` 是 Qt 版的 `printf`，用于在控制台打印调试信息。
*   **工业级认知**:
    1.  `qDebug()` 是开发期的“X光机”。
    2.  在 `.pro` 文件里加 `CONFIG += console` 会在发布时弹出一个丑陋的黑框，**绝对不能用于最终交付给客户的商业软件**。工业级日志记录用的是 `qInstallMessageHandler` 重定向到文件。

**第6-8课: QByteArray, QString**
*   **核心知识点**:
    1.  `QByteArray` 处理原始字节流，`QString` 处理带编码的文本。
    2.  API 极其相似，都有 `append`, `replace`, `size` 等。
    3.  `QString::arg()` 可以安全、便捷地格式化字符串。
*   **工业级认知**:
    1.  **定位**: `QByteArray` 是给**机器**看的“快递箱”（网络/硬件/文件IO）；`QString` 是给**人类**看的“精装记事本”（UI显示/多语言）。
    2.  **尺寸陷阱 (面试必考)**: `QString::size()` 计算的是“个数”（`"我爱Qt"` 是4个）；`QByteArray::size()` 计算的是“字节重量”（UTF-8下是8个字节）。搞混会导致网络发包截断、内存申请错误等严重 Bug。
    3.  **边界桥梁**: `QString` 转 `QByteArray` 用 `.toUtf8()`。`QString` 送去给纯 C 函数用，终极连招是 **`str.toUtf8().data()`**。

**第9-11课: QVariant**
*   **核心知识点**:
    1.  `QVariant` 是一个可以存储任意 Qt 已知类型的“万能盒子”。
    2.  用 `Q_DECLARE_METATYPE` 宏可以让你自定义的结构体也能被 `QVariant` 存储。
*   **工业级认知**:
    1.  **反面教材**: **绝对不要**像视频里那样用 `QVariant` 去代替 C++ 的函数重载来写加法器，这是极其糟糕的设计。
    2.  **真正用途**: `QVariant` 是 Qt 架构在**“系统边界”**上的“标准集装箱”，用于 `QSettings` (配置文件)、数据库 (`SQL`)、模型/视图 (`Model/View`) 等无法在编译期确定数据类型的场景。
    3.  **注册机制**: `Q_DECLARE_METATYPE` 只是给自定义类型办了张“护照”，让 `QVariant` 认识它。

#### **第二部分：核心交互与事件 (The Nervous System)**

**第12-16课: QTimer, 信号槽, QElapsedTimer**
*   **核心知识点**:
    1.  信号槽是 Qt 的核心通信机制，`connect` 函数用于连接。
    2.  `QTimer` 通过周期性发射 `timeout` 信号来驱动事件。`singleShot` 用于延时单次触发。
    3.  `QElapsedTimer` 用于精确测量代码耗时。
*   **工业级认知**:
    1.  `connect` 是“定闹钟”（注册回调），而非“立即执行”。现代 C++ 语法 `&Class::signal` 有编译期类型检查，**绝对淘汰**了 Qt4 的 `SIGNAL()` 宏。
    2.  **重载歧义**: 当信号有重载时，必须使用 **`qOverload`** 模板来消除歧义，而不是用视频里丑陋的 C++98 函数指针。
    3.  **UI 线程陷阱**: `QTimer` 的槽函数在主线程执行，**绝对不能有耗时操作**，否则整个界面会卡死。
    4.  **精确计时**: `QTime` 是“挂钟”，会受系统时间修改影响；`QElapsedTimer` 是“秒表”，基于 CPU 时钟，是性能测试的唯一标准。

#### **第三部分：标准对话框与UI实践 (The Toolbox & Workshop)**

**第17-21课: UI文件, QWidget API, 右键菜单**
*   **核心知识点**:
    1.  `.ui` 文件是 XML，会被 `uic` 工具翻译成 `ui_xxx.h` 的 C++ 代码。
    2.  `QWidget` 有大量 API 用于设置尺寸、位置、图标等。
    3.  通过 `setContextMenuPolicy` 和 `customContextMenuRequested` 信号可以实现右键菜单。
*   **工业级认知**:
    1.  `ui_xxx.h` 是自动生成的文件，**绝对不能手动修改**。
    2.  **不要背 API**！依赖 IDE 的代码自动补全。唯一需要记牢的概念是 `geometry()` (套内面积) vs `frameGeometry()` (建筑面积)。
    3.  **右键转到槽的巨坑**: `on_按钮名_clicked()` 这种自动关联机制，一旦在 UI 修改按钮名，代码会**无报错地“静默失效”**，是项目维护的灾难。**永远手动写 `connect`**。
    4.  **坐标系陷阱**: 右键菜单的 `pos` 是局部坐标，但 `QMenu::exec()` 需要全局坐标，标准做法是用 `this->mapToGlobal(pos)` 进行转换。

**第22-32课: 标准对话框全家桶 (`QDialog` 到 `QProgressDialog`)**
*   **核心知识点**:
    1.  `QDialog::exec()` 的返回值 `Accepted/Rejected` 是模态交互的核心。
    2.  `QMessageBox`, `QFileDialog`, `QFontDialog`, `QColorDialog`, `QInputDialog` 提供了极其方便的静态方法，一行代码解决问题。
    3.  `QProgressDialog` 需要配合 `QTimer` 或多线程来更新进度。
*   **工业级认知**:
    1.  **自定义对话框架构**: 将复杂的弹窗（如登录、设置）封装在独立的 `QDialog` 子类中，实现高内聚。主窗口只管调用 `exec()` 并判断返回值。
    2.  **信号反模式**: 用 `exec()` 时，**绝对不要**去监听 `accepted/rejected` 信号，这是画蛇添足。
    3.  **布局大法**: 任何需要自适应的界面，都必须用**布局管理器**，而不是 `move`。居中靠“弹簧”(`addStretch`)。
    4.  **文件保存错觉**: `QFileDialog::getSaveFileName` **只返回一个字符串路径**，并不会创建或保存文件。真正的写入操作需要 `QFile` 和 `QTextStream`。
    5.  **进度条内存灾难**: 视频中在槽函数里用 `static` 变量和 `delete` 指针是**极其危险的错误示范**。安全做法是使用 `setAttribute(Qt::WA_DeleteOnClose)` 让窗口自杀，或用 `deleteLater()`。

**第33-40课: QMainWindow 深度实践与布局**
*   **核心知识点**:
    1.  `QMainWindow` 有固定的菜单栏、工具栏、状态栏、浮动窗口、核心区。
    2.  `QAction` 是菜单项和工具栏按钮的载体。
    3.  `QToolBar` 可以用 `addWidget` 添加任意控件。
    4.  `QStatusBar` 有 `showMessage` (临时) 和 `addPermanentWidget` (永久) 两种方式。
    5.  `QDockWidget` 用于创建可停靠的浮动面板。
    6.  `.qrc` 资源文件可以将图片等打包进 `.exe`。
    7.  布局有“外边距(Margin)”和“内间距(Spacing)”。
*   **工业级认知**:
    1.  **命令模式 (`QAction`)**: `QAction` 是“万能遥控器按键”，一个 `Action` 可以同时放进菜单栏、工具栏，并绑定快捷键，实现一处修改，处处生效。
    2.  **快捷键标准**: 快捷键要用 `QKeySequence::Save` 这样的标准枚举，而不是写死 `"Ctrl+S"`，以保证跨平台兼容。
    3.  **状态栏架构**: `showMessage` 的临时消息会自动隐藏和恢复常驻控件，**绝对不需要**像视频里那样用定时器手动 `show()`。
    4.  **登录拦截架构**: 在 `main.cpp` 中先 `exec()` 登录对话框，成功后再创建和显示 `MainWindow`，这是所有桌面软件的标准安全启动流程。
    5.  **布局精髓**: 嵌套布局时，子布局的 `Margin` 必须清零，否则边距会丑陋地叠加。
