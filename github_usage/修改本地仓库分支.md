## 问  
怎么切换本地仓库的分支，比如从master切换为main  

## 答  
要把本地仓库分支从 `master` 切换（重命名）为 `main`，请参考以下步骤：

---

### **1. 查看当前分支**

```sh
git branch
```
如果你当前在 `master`，你应该会看到：
```
* master
```

---

### **2. 重命名分支 master 为 main**

```sh
git branch -m master main
```
如果现在就已经在 master 上，直接运行这个命令即可。

---

### **3. 推送新分支到远程仓库并建立追踪关系**

如果远程还没有 `main` 分支：

```sh
git push -u origin main
```

---

### **4. （可选）删除远程的 master 分支**

如果已经确认你的代码都在 `main` 并且主分支应该叫 `main`，你可以删除远程的 `master`（小心使用！）：

```sh
git push origin --delete master
```

---

### **5. 更新仓库默认分支（GitHub 页面上）**

到你的 GitHub 仓库**Settings → Branches**，将默认分支改为 `main`。  
然后可以删除旧的 `master` 分支。

---

#### **小结**

- 本地分支改名：`git branch -m master main`
- 新分支上传远程：`git push -u origin main`
- 如有需要再删除远程 master：`git push origin --delete master`

有任何步骤报错，把报错内容粘贴我帮你分析！

## 如何切换到已存在的分支

如果你要切换到本地仓库中已经存在的其他分支（不是重命名，而是切换分支），请使用以下方法：

---

### **1. 查看所有分支**

查看本地所有分支：
```sh
git branch
```

查看远程所有分支：
```sh
git branch -r
```

查看本地和远程所有分支：
```sh
git branch -a
```

---

### **2. 切换到已存在的本地分支**

```sh
git checkout 分支名
```

例如，切换到 `develop` 分支：
```sh
git checkout develop
```

或者使用更新的命令（Git 2.23+）：
```sh
git switch develop
```

---

### **3. 切换到远程分支**

如果你要切换到一个只在远程存在的分支，Git 会自动创建本地追踪分支：

```sh
git checkout 分支名
```

例如，远程有 `origin/feature-branch`，运行：
```sh
git checkout feature-branch
```

Git 会自动创建本地的 `feature-branch` 分支并追踪远程的 `origin/feature-branch`。

---

### **4. 显式创建本地分支追踪远程分支**

```sh
git checkout -b 本地分支名 origin/远程分支名
```

例如：
```sh
git checkout -b develop origin/develop
```

---

### **5. 查看当前分支**

```sh
git branch
```

输出中带 `*` 号的分支就是你当前所在的分支。

---

#### **常用快速切换**

- 切换到上一个分支：`git checkout -`
- 查看分支详细信息：`git branch -v`
- 查看分支的追踪关系：`git branch -vv`
