你不需要懂 git 内部原理，只照着打就行。

## 🧭 目标

保留你现在仓库 ProcLab 的所有历史代码，把它「封存」到一个单独分支 `archive/2025-11-03-before-reset` 里，
然后在当前分支（例如 `main`）上重新开始干净的练手。

---

## 🪜 步骤（从零开始）

> ⚠️ 下面命令都在你电脑的 **终端（不是 GitHub 网页）** 里执行。
> 如果你没 clone 仓库，可以先用第 0 步；如果本地已经有 ProcLab 目录，就从第 1 步开始。

---

### 0️⃣ 先把仓库拉到本地（如果还没有）

```bash
git clone git@github.com:whytreatme/ProcLab.git
cd ProcLab
```

---

### 1️⃣ 确认你在主分支（一般叫 main 或 master）

```bash
git branch
```

看到类似：

```
* main
```

就表示当前在 main 分支。如果不是 main 而是 master ，那下面命令的 main 都换成 master 。

---

### 2️⃣ 创建一个封存分支

```bash
git checkout -b archive/2025-11-03-before-reset
```

含义：从你当前的全部代码状态创建一个名叫 `archive/2025-11-03-before-reset` 的新分支。
成功后终端会显示：

```
Switched to a new branch 'archive/2025-11-03-before-reset'
```

---

### 3️⃣ 把这个分支推送到 GitHub 上保存

```bash
git push -u origin archive/2025-11-03-before-reset
```

成功后你会看到一堆上传进度，最后类似：

```
Branch 'archive/2025-11-03-before-reset' set up to track remote branch 'archive/2025-11-03-before-reset' from 'origin'.
```

这一步执行完，GitHub 上已经多了一个封存分支，你的旧项目完全保留。

---

### 4️⃣ 回到 main 分支，准备重新开始

```bash
git checkout main
```

### 5️⃣ 清空旧文件（不删历史）

```bash
git rm -rf .
git clean -fdx
```

### 6️⃣ 新建最小目录

```bash
mkdir -p src bin
touch src/main.cpp Makefile
git add .
git commit -m "reset: start clean practice version"
git push
```

这样，`main` 分支就干净了，可以直接放我给你的最简练手代码。

---

### ✅ 结果检查

到 GitHub 网页上看：

* “Branches” 里多了一个 `archive/2025-11-03-before-reset` ——那是你的旧项目；
* “Default branch” 仍是 main，它现在是空壳，准备写新的练手内容。

我懂了，创建一个新的archieve分支就是从当前main中分支出去，然后把main中的相关记录清空，就得到了一个干净的main分支和保存原来内容的archieve分支  

完全正确 ✅！你理解得非常到位。

你现在掌握的意思就是 Git 的本质操作逻辑：

---

### 🧩 一句话概括

> 创建 `archive/...` 分支，是从当前 `main` 的代码状态复制出去保存；
> 然后在 `main` 上“清空工作区、重新提交”，就得到了一个干净的起点。

---

### 🧠 用图来形容更直观：

```
       （现在）
        main ──────●───●───●───●───┐
                                          ↓
                           git checkout -b archive/2025-11-03-before-reset
                                          ↓
archive/2025-11-03-before-reset ─────────●───●───●───●───(保留旧历史)

        main ──(清空重新提交)───○───(干净起点)
```

* `archive/...` 分支：
  → 100% 保存你原来的提交、代码、日志、实验痕迹。
* `main` 分支：
  → 你现在重新开始写练手版本，从空目录起步。

---

### 💡 这样做的好处

1. **安全**：旧内容完全保留在 `archive`，一点不丢；
2. **干净**：新的 main 不会混乱；
3. **灵活**：随时可以切回旧项目继续参考（`git checkout archive/...`）。

---

你现在只差一步：

> 在 `main` 分支里删掉旧文件后 `commit` 一下，就是你的“干净版本起点”。
