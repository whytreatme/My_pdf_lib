## 一、你不喜欢自动补全 → Copilot 该怎么用才不反学习？

### 先说结论

👉 **你现在用 Copilot 的正确姿势是：只用 Chat，不用 Inline 自动补全。**

### 正确设置（很关键）

在 VS Code / JetBrains 里：

* 关掉：

  * 自动补全（Inline Suggestions）
  * Tab 自动接收整段代码
* 保留：

  * **Copilot Chat**
  * 右键 → “Explain this code”
  * 右键 → “Suggest refactor / tests”

这样 Copilot 变成了：

> **“随叫随到的代码助教”，而不是“替你写代码的外挂”**

### 你可以这样用（非常适合初学者）

举几个你会真的用上的场景：

* **你自己先写一版 API / 类**

  * 再问 Copilot：

    > 这个接口设计在工程上有没有明显问题？
* **你卡在某个 POSIX / STL / Reactor 细节**

  * 问：

    > epoll 的 EPOLLONESHOT 在这个场景下会出什么坑？
* **你写完 Buffer / Connection**

  * 让它：

    > 帮我找可能的内存生命周期问题

📌 **重点：你先写，AI 后看。**
这是高手路径，不是菜鸟路径。

## 二、模型差异：不讲参数，只讲“工程感受”

下面我不讲营销词，只讲**真实使用手感**。

---

### 🧠 ChatGPT（GPT-5.x 系）

**气质：全能工程思考型**

**优点**

* 工程抽象能力强（模块划分、架构解释）
* 特别适合：

  * 学习一个新概念
  * 把“模糊理解”讲清楚
  * 解释“为什么这么设计”

**缺点**

* 不在 IDE 里
* 写代码“稳”，但不快
* 偶尔会过度抽象

**最适合你什么时候用？**

* 学 Reactor / 网络编程 / OS / 数据结构
* 想搞清楚：

  > 这个设计到底在解决什么问题？

👉 **这是你的“主学习脑”**

---

### 🧑‍💻 GitHub Copilot（本质是“工程上下文 AI”）

**气质：工程现场助理**

**优点**

* 能“看到你的整个工程”
* 非常擅长：

  * 补全重复代码
  * 写测试
  * 小范围重构
* Chat 模式对代码上下文理解非常好

**缺点**

* 哲学能力弱
* 不擅长从 0 教你一个概念
* 自动补全容易破坏学习节奏（你已经意识到了 👍）

**最适合你什么时候用？**

* **工程阶段**
* 你已经知道“要写什么”，只是：

  * 写得慢
  * 怕漏边界条件

👉 **这是你的“工程副手”**

---

### 🌐 Gemini（Google 系）

**气质：搜索 + 文档理解型**

**优点**

* 非常擅长：

  * 查资料
  * 对比不同 API
  * 官方文档总结
* 对 Web / 云 / Android / Google 技术栈友好

**缺点**

* 代码“工程感”偏弱
* 系统级、底层设计不够老练

**适合场景**

* 快速了解一个库
* 查：

  > std::chrono vs clock_gettime 区别

👉 **这是“技术搜索引擎升级版”**

---

### 🪶 Claude（Anthropic）

**气质：谨慎、整洁、解释型**

**优点**

* 代码风格干净
* 特别擅长：

  * 长代码 review
  * 找 bug
  * 写文档和注释

**缺点**

* 性能、系统编程理解略弱
* 偏保守，有时不敢给激进方案

**适合场景**

* review 自己写的复杂代码
* 整理工程说明文档

👉 **这是“严谨型审稿人”**

---

## 三、站在“学习者 + 工程开发者”的最优组合

我直接给你一个**不烧钱、不反学习**的方案：

### ✅ 推荐组合（非常适合你）

* **ChatGPT（Free 或 Plus）**
  👉 用来：

  * 学概念
  * 纠正理解模型
  * 架构层面的讨论

* **Copilot Pro（$10）但：**

  * ❌ 不用自动补全
  * ✅ 只用 Chat / Review / Explain

这个组合的核心思想是：

> **“我思考，我写；AI 提醒我盲点。”**
