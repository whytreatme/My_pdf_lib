## 问题一
$ git push -u origin main
To https://github.com/whytreatme/BankApp.git
 ! [rejected]        main -> main (non-fast-forward)
error: failed to push some refs to 'https://github.com/whytreatme/BankApp.git'
hint: Updates were rejected because the tip of your current branch is behind
hint: its remote counterpart. If you want to integrate the remote changes,
hint: use 'git pull' before pushing again.
hint: See the 'Note about fast-forwards' in 'git push --help' for details.

h@MyComputer MINGW64 /e/mypdf_local_repo/Qt/code (main)
$ git pull
There is no tracking information for the current branch.
Please specify which branch you want to merge with.
See git-pull(1) for details.

    git pull <remote> <branch>

If you wish to set tracking information for this branch you can do so with:

    git branch --set-upstream-to=origin/<branch> main
怎么回事？  

## 回答一  
你的问题是：  
- `git push` 报错：`[rejected] main -> main (non-fast-forward)`  
- `git pull` 又提示没有 tracking 信息

**原因：**  
这表示你的远程 `main` 分支和本地 `main` 分支存在差异（如远程有提交，本地没有），而且本地没有设置追踪（tracking）远程分支，导致无法自动合并。

---

### 解决办法

#### 1. **设置 upstream 并拉取远程变动**

先让你的本地 main 跟踪远程 main，然后拉取远程 main 的最新内容与你本地合并：

```sh
git branch --set-upstream-to=origin/main main
git pull
```

拉取时可能会遇到合并冲突，如果有需按提示解决。

---

#### 2. **合并/解决冲突后再推送**

如果上面步骤拉取成功，或已解决冲突，可以再 push：

```sh
git push -u origin main
```

---

#### ⚠️ 说明

如果**你确认远程 main 分支可以被你本地 main 覆盖**，且不关心历史，可以强制推送（慎用，会覆盖远程）：

```sh
git push -u origin main --force
```
> 只有当你确定远程 main 的内容都可以被你本地 main 覆盖时才强行推送，一般团队开发不建议直接强制 push！

---

### 总结

1. 设 tracking 并拉取：  
   ```sh
   git branch --set-upstream-to=origin/main main
   git pull
   ```
2. 解决冲突后 push：  
   ```sh
   git push -u origin main
   ```
3. 如需强制覆盖远程：  
   ```sh
   git push -u origin main --force
   ```

## 问题二  
h@MyComputer MINGW64 /e/mypdf_local_repo/Qt/code (main)
$   git branch --set-upstream-to=origin/main main
branch 'main' set up to track 'origin/main'.

h@MyComputer MINGW64 /e/mypdf_local_repo/Qt/code (main)
$ git pull
fatal: refusing to merge unrelated histories
怎么回事？  

## 回答二  
你遇到的报错：

```
fatal: refusing to merge unrelated histories
```

**原因说明：**  
这是因为你的本地仓库和远程仓库的历史完全无关（比如你新建本地仓库、也新建远程仓库，各自有独立初始化，没有共同的祖先 commit）。Git 默认不会强行合并两条无关的提交历史，所以会报这个错。

---

### 解决办法

如果你**确定本地和远程合并是安全的**，可以加 `--allow-unrelated-histories` 参数来拉取：

```sh
git pull origin main --allow-unrelated-histories
```

合并过程中如有冲突，手动解决冲突后提交，再正常 push：

```sh
git add .
git commit -m "fix: merge unrelated histories and resolve conflicts"
git push -u origin main
```

---

### 总结步骤

1. 强制拉取合并远程代码并允许无关历史：

    ```sh
    git pull origin main --allow-unrelated-histories
    ```

2. 如果产生冲突，手动编辑文件解决冲突：

    - 用 `git status` 查看冲突文件
    - 手动编辑文件解决冲突后：
      ```sh
      git add .
      git commit -m "resolve conflict"
      ```

3. 合并完成后重新推送：

    ```sh
    git push -u origin main
    ```

---

**建议：**  
- 这是两个仓库历史不相关的常见情况，例如：你本地初始化时有自己的 commit，远程仓库也有 README 或 LICENSE 等初始化文件。
- 合并完以后，历史就唯一了，后面操作就没有这类问题了。

  
## 问题三    
h@MyComputer MINGW64 /e/mypdf_local_repo/Qt/code (main)
$ git pull origin main --allow-unrelated-histories
From https://github.com/whytreatme/BankApp
 * branch            main       -> FETCH_HEAD
error: Your local changes to the following files would be overwritten by merge:
  .gitattributes
Merge with strategy ort failed.
这个呢？    

你遇到的错误：

```
error: Your local changes to the following files would be overwritten by merge:
  .gitattributes
Merge with strategy ort failed.
```

**原因：**  
你的本地 `.gitattributes` 文件已经有了尚未提交的更改，拉取（pull）时会被远程分支的内容覆盖，Git 为了保护你的本地更改，不会自动合并。

---

## **如何解决？有三种方法：**

### 方法一：**先提交本地更改再拉取**

1. **添加并提交你的当前更改**
   ```sh
   git add .
   git commit -m "临时提交：为拉取远程分支保存本地更改"
   ```

2. **拉取并合并**
   ```sh
   git pull origin main --allow-unrelated-histories
   ```
   > 如遇冲突按提示解决。

---

### 方法二：**暂存本地更改（Stash）**

1. **把本地改动先存起来**
   ```sh
   git stash
   ```

2. **拉取远程内容合并**
   ```sh
   git pull origin main --allow-unrelated-histories
   ```

3. **恢复你的本地更改**
   ```sh
   git stash pop
   ```

4. 如有冲突，解决后继续 `git add .`、`git commit -m "合并冲突"`。

---

### 方法三：**放弃本地未提交的更改（慎用！）**
如果你确认本地 `.gitattributes` 的更改无关紧要，可以直接放弃本地更改：

```sh
git checkout -- .gitattributes
```

然后再拉取：

```sh
git pull origin main --allow-unrelated-histories
```

---

## **推荐做法**
通常推荐**方法一**或**方法二**，避免误删重要本地内容。
