# 问  
现实中那些代码需要进行压力测试？  

# 答  

在实际生产环境中，以下类型的代码或系统通常需要进行压力测试：

1.  **高并发 Web 接口**：如电商秒杀系统、热门抢票系统、社交媒体的动态流。
2.  **数据库操作**：涉及大量并发读写、复杂事务处理或深分页查询的代码。
3.  **实时通信服务**：如即时通讯（IM）、在线游戏后端、直播弹幕系统。
4.  **金融级系统**：支付网关、高频交易撮合引擎，需确保在极端负载下的数据一致性和低延迟。
5.  **中间件与基础组件**：消息队列（Kafka/RabbitMQ）的生产者与消费者、自定义的 RPC 框架。
6.  **分布式系统边界**：负载均衡器配置、服务发现逻辑、断路器（Circuit Breaker）触发阈值。
7.  **大数据处理链路**：ETL 流程、流式处理（Flink/Spark Streaming）的数据摄入点。

压力测试的核心目的是寻找系统的**性能瓶颈**（如 CPU 耗尽、内存泄漏、数据库锁竞争或 I/O 阻塞）以及确定系统的**崩溃临界点**。
